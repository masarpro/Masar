## 1. نظرة عامة على القسم

قسم التشطيبات هو أحد أقسام دراسة الكميات (CostStudy) في تطبيق مسار. يتكون من:

- **بطاقة إعدادات المبنى** (BuildingConfigPanel) - في أعلى الصفحة
- **ملخص التشطيبات** (FinishingSummary) - عدد البنود والتكلفة الإجمالية
- **قوالب سريعة** (QuickAddTemplates) - توليد بنود تلقائية حسب مستوى التشطيب
- **26 فئة تشطيب** موزعة على **15 مجموعة عمل**

### المسار في التطبيق
```
/app/[organizationSlug]/pricing/studies/[studyId]/finishing
```

## 2. بطاقة إعدادات المبنى - الوضع الحالي

### البيانات المخزنة (buildingConfig في CostStudy كـ JsonB)

```typescript
interface BuildingConfig {
  totalLandArea: number;      // مساحة الأرض الإجمالية (م²)
  buildingPerimeter: number;  // محيط المبنى (م.ط)
  floors: FloorConfig[];      // مصفوفة الأدوار
}

interface FloorConfig {
  id: string;           // معرف فريد
  name: string;         // اسم الدور (بدروم، أرضي، أول، ...)
  area: number;         // مساحة الدور (م²)
  height: number;       // ارتفاع الدور (م)
  sortOrder: number;    // ترتيب العرض
  isRepeated: boolean;  // هل الدور مكرر؟
  repeatCount: number;  // عدد التكرار
  floorType: FloorType; // نوع الدور
}

// أنواع الأدوار مع الارتفاعات الافتراضية
enum FloorType {
  BASEMENT   = 3.0m
  GROUND     = 3.2m
  UPPER      = 3.2m
  ANNEX      = 3.0m
  ROOF       = 0.0m
  MEZZANINE  = 2.8m
}
```

### الاستخدام الحالي لإعدادات المبنى

حالياً، إعدادات المبنى تُستخدم بشكل **محدود** في:

1. **اختيار الدور** (FloorSelector): عند إضافة بند بنطاق `PER_FLOOR`، يظهر قائمة بالأدوار المعرّفة في الإعدادات لاختيار الدور المطلوب.
2. **القوالب السريعة** (QuickAddTemplates): عند توليد بنود تلقائية، تُستخدم مساحات الأدوار وارتفاعاتها لحساب الكميات الأولية.
3. **مساحة الحوش**: تُحسب كـ `مساحة الأرض - مساحة المبنى` في القوالب.

### المشكلة الحالية
**لا يوجد ربط ديناميكي** بين إعدادات المبنى وبنود التشطيبات الفردية. كل بند يُدخل مساحاته وأبعاده يدوياً ومستقلاً عن الآخرين. هذا يعني:
- تكرار إدخال نفس المساحات والأبعاد في كل بند
- عدم تحديث البنود تلقائياً عند تغيير إعدادات المبنى
- عدم استفادة البنود من بيانات بعضها البعض

## 4. طرق الحساب المتاحة

### 4.1 المساحة المباشرة (DIRECT_AREA)
- إدخال المساحة مباشرة أو الطول × العرض
- **يُستخدم في:** العزل المائي، العزل الحراري، أرضيات الحوش، تنسيق الحدائق، تشطيبات السطح

### 4.2 خصم الفتحات (WALL_DEDUCTION)
- محيط الجدران × الارتفاع - مساحات الفتحات (أبواب ونوافذ)
- خيار إضافة السقف
- فتحات مسبقة: باب عادي (0.9×2.1)، باب حمام (0.7×2.1)، باب رئيسي (1.2×2.4)، نافذة كبيرة (1.5×1.5)، نافذة متوسطة (1.2×1.2)، نافذة صغيرة (0.6×0.6)
- **يُستخدم في:** اللياسة الداخلية/الخارجية، الدهان، تكسيات الجدران، الواجهات الحجرية

### 4.3 غرفة بغرفة (ROOM_BY_ROOM)
- إضافة غرف فردية باسم وطول وعرض
- حساب المساحة الإجمالية كمجموع مساحات الغرف
- **يُستخدم في:** الأرضيات، الأسقف المستعارة

### 4.4 لكل وحدة (PER_UNIT)
- إدخال العدد فقط
- **يُستخدم في:** الأبواب، النوافذ، الحمامات، بوابات السور

### 4.5 خطي (LINEAR)
- إدخال الأمتار الطولية
- **يُستخدم في:** المطابخ، الدرابزين، الدرج

### 4.6 مقطوعية (LUMP_SUM)
- سعر مواد + سعر عمالة = الإجمالي
- **يُستخدم في:** زخارف الواجهات، الديكورات الداخلية

## 5. خريطة العلاقات والتكرار بين البنود

هذا القسم يوضح العلاقات المنطقية بين البنود التي تتشارك في نفس الكميات أو يمكن اشتقاق إحداها من الأخرى.

### 5.1 مجموعة علاقات السطح

```
┌─────────────────────────────────────────────────────┐
│                   مساحة السطح                        │
│            (= مساحة آخر دور متكرر)                   │
│                                                      │
│    ┌──────────┐   ┌──────────────┐   ┌────────────┐ │
│    │ العزل    │ = │ العزل        │ = │ بلاط       │ │
│    │ المائي   │   │ الحراري      │   │ السطح      │ │
│    │ (سطح)    │   │ (سطح)        │   │            │ │
│    └──────────┘   └──────────────┘   └────────────┘ │
│                                                      │
│    ┌──────────────────────────────────────────────┐  │
│    │ طول دروة السطح (محيط السطح)                  │  │
│    │ = وزرات السطح (م.ط) = درابزين السطح (م.ط)   │  │
│    └──────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
```

**التفسير:**
- مساحة العزل المائي للسطح = مساحة العزل الحراري للسطح = مساحة بلاط السطح
- طول دروة السطح (محيط) = طول وزرات السطح = طول درابزين السطح
- كل هذه القيم يمكن اشتقاقها من مساحة ومحيط آخر دور (UPPER أو GROUND)

### 5.2 مجموعة علاقات اللياسة ↔ الدهانات

```
┌───────────────────────────────────────────────────────────┐
│                    لكل دور على حدة                        │
│                                                           │
│  ┌─────────────────┐                ┌──────────────────┐  │
│  │ اللياسة الداخلية │ ─── نفس ───→ │ الدهان الداخلي   │  │
│  │ (جدران + سقف)   │   الكمية     │ (باستثناء        │  │
│  │                  │              │  جدران الحمامات)  │  │
│  └────────┬─────────┘              └──────────────────┘  │
│           │                                               │
│           │ الفتحات المستثناة                              │
│           ▼                                               │
│  ┌─────────────────┐                                      │
│  │ الأبواب والنوافذ │ ← نفس الفتحات المخصومة              │
│  │ (عدد ومساحات)   │   من اللياسة/الدهان                  │
│  └─────────────────┘                                      │
│                                                           │
│  ┌─────────────────┐                ┌──────────────────┐  │
│  │ جدران الحمامات  │ ─── نفس ───→ │ بلاط جدران      │  │
│  │ (غير مدهونة)    │   الكمية     │ الحمامات          │  │
│  └─────────────────┘              └──────────────────┘  │
└───────────────────────────────────────────────────────────┘
```

**التفسير:**
- مساحة الدهان الداخلي = مساحة اللياسة الداخلية - مساحة جدران الحمامات
- مساحة جدران الحمامات التي لا تُدهن = مساحة بلاط جدران الحمامات (تكسيات)
- الفتحات (أبواب + نوافذ) المخصومة من اللياسة = نفس الفتحات المخصومة من الدهان = نفس عدد ومساحات الأبواب والنوافذ

```
│  محيط السور = محيط الأرض (م.ط)                            │
│  ┌──────────────────┐   ┌──────────────────┐             │
│  │ دهان السور       │   │ لياسة السور      │             │
│  │ (محيط × ارتفاع   │   │ (نفس المساحة)    │             │
│  │  السور × 2       │   │                  │             │
│  │  وجهين)          │   │                  │             │
│  └──────────────────┘   └──────────────────┘             │
└───────────────────────────────────────────────────────────┘
```

## 6. التحليل التفصيلي: تأثير إعدادات المبنى على كل بند

### جدول الاشتقاق من إعدادات المبنى

| البند | ما يمكن اشتقاقه من إعدادات المبنى | المعادلة |
|-------|--------------------------------------|----------|
| **العزل المائي (أساسات)** | مساحة دور البدروم | `floors.find(BASEMENT).area` |
| **العزل المائي (سطح)** | مساحة السطح | `floors.find(ROOF).area` أو مساحة آخر دور |
| **العزل المائي (حمامات)** | يحتاج بيانات إضافية | عدد الحمامات × متوسط مساحة الحمام |
| **العزل الحراري (جدران)** | محيط المبنى × مجموع الارتفاعات | `buildingPerimeter × Σ(floor.height × repeatCount)` |
| **العزل الحراري (سطح)** | مساحة السطح | `= عزل مائي السطح` |
| **اللياسة الداخلية** | لكل دور: محيط الغرف × ارتفاع | `floor.area → محيط مشتق × floor.height - فتحات` |
| **اللياسة الخارجية** | محيط المبنى × الارتفاع الكلي | `buildingPerimeter × Σ(heights) - فتحات خارجية` |
| **الدهان الداخلي** | = اللياسة الداخلية - جدران الحمامات | `لياسة_داخلية - بلاط_جدران_حمامات` |
| **دهان الواجهات** | = اللياسة الخارجية | `= لياسة_خارجية` |
| **دهان السور** | محيط الأرض × ارتفاع السور × 2 | يحتاج حقل إضافي: محيط الأرض |
| **الأرضيات** | مساحة كل دور | `floor.area` (تقريباً) |
| **تكسيات الجدران** | جدران الحمامات | القيمة المستثناة من الدهان |
| **الأسقف المستعارة** | مساحة كل دور | `floor.area` أو مساحة الغرف المحددة |
| **أرضيات الحوش** | مساحة الأرض - مساحة المبنى | `totalLandArea - floors.find(GROUND).area` |
| **تشطيبات السطح** | مساحة السطح | `floors.find(ROOF).area` |
| **الواجهات الحجرية** | = اللياسة الخارجية | `= لياسة_خارجية` (أو جزء منها) |

### القيم المشتقة من إعدادات المبنى

```
من مساحة الأرض (totalLandArea):
  ├── مساحة الحوش = totalLandArea - مساحة الدور الأرضي
  ├── أرضيات الحوش
  └── تنسيق الحدائق

من محيط المبنى (buildingPerimeter):
  ├── اللياسة الخارجية = محيط × ارتفاع كلي
  ├── دهان الواجهات = لياسة خارجية
  ├── الواجهات الحجرية = لياسة خارجية (أو نسبة منها)
  └── العزل الحراري للجدران = محيط × ارتفاع كلي

من كل دور (floor):
  ├── مساحة الأرضيات ≈ مساحة الدور
  ├── مساحة الأسقف ≈ مساحة الدور
  ├── اللياسة الداخلية = f(مساحة الدور, ارتفاع الدور, فتحات)
  ├── الدهان الداخلي = لياسة داخلية - جدران حمامات
  └── التكرار: isRepeated × repeatCount يضاعف كل الكميات

من دور السطح (ROOF):
  ├── العزل المائي للسطح
  ├── العزل الحراري للسطح
  ├── بلاط السطح
  ├── وزرات السطح = محيط السطح
  └── درابزين السطح = محيط السطح (أو جزء)
```

## 7. مصفوفة الربط المقترحة بين البنود

### 7.1 علاقات التساوي المباشر (A = B)

| المصدر (A) | الهدف (B) | العلاقة | ملاحظات |
|-------------|-----------|---------|---------|
| العزل المائي (سطح) | العزل الحراري (سطح) | تساوي كامل | نفس مساحة السطح |
| العزل المائي (سطح) | بلاط السطح | تساوي كامل | نفس مساحة السطح |
| اللياسة الداخلية | الدهان الداخلي | تساوي مع استثناء | الدهان = اللياسة - جدران الحمامات |
| اللياسة الخارجية | دهان الواجهات | تساوي كامل | نفس المساحة الخارجية |
| اللياسة الخارجية | الواجهات الحجرية | تساوي أو نسبة | قد يكون الحجر جزء من الواجهة |
| أسقف اللياسة (لكل دور) | الأسقف المستعارة | تساوي كامل | نفس مساحات الغرف |
| جدران حمامات (غير مدهونة) | بلاط جدران الحمامات | تساوي كامل | ما يُستثنى من الدهان = ما يُبلّط |
| فتحات اللياسة الداخلية | الأبواب والنوافذ | تساوي كامل | الفتحات المخصومة = الأبواب والنوافذ الفعلية |
| محيط السطح (دروة) | وزرات السطح | تساوي كامل | نفس الطول |

### 7.2 علاقات الاشتقاق (A → B بعملية حسابية)

| المصدر (A) | الهدف (B) | العملية | المعادلة |
|-------------|-----------|---------|----------|
| مساحة الأرض - مساحة المبنى | أرضيات الحوش | طرح | `totalLandArea - buildingArea` |
| محيط المبنى × ارتفاعات الأدوار | اللياسة الخارجية | ضرب | `perimeter × Σ(heights)` |
| مساحة الدور | الأرضيات (تقريبي) | مباشر | `floor.area` |
| مساحة الدور | الأسقف (تقريبي) | مباشر | `floor.area` |

### 7.3 مخطط سلسلة الاشتقاق الكاملة

```
إعدادات المبنى
│
├── totalLandArea ──────────────────────────────────────────┐
│                                                           │
├── buildingPerimeter ──┬── × Σ(heights) ──→ لياسة خارجية ─┤
│                       │                    ├── = دهان واجهات
│                       │                    └── = واجهات حجرية
│                       │
│                       └── × ارتفاع كلي ──→ عزل حراري جدران
│
├── floor[GROUND]
│   ├── area ──→ أرضيات أرضي
│   │           ├── = أسقف مستعارة أرضي
│   │           └── مع totalLandArea → مساحة الحوش
│   ├── height ──→ لياسة داخلية أرضي
│   │              ├── - حمامات = دهان أرضي
│   │              └── الفتحات = أبواب + نوافذ أرضي
│   └── area (حمامات) ──→ بلاط جدران حمامات أرضي
│
├── floor[UPPER] (× repeatCount)
│   ├── area ──→ أرضيات الدور
│   │           └── = أسقف مستعارة الدور
│   ├── height ──→ لياسة داخلية الدور
│   │              ├── - حمامات = دهان الدور
│   │              └── الفتحات = أبواب + نوافذ الدور
│   └── area (حمامات) ──→ بلاط جدران حمامات الدور
│
├── floor[ROOF]
│   ├── area ──→ عزل مائي سطح
│   │           ├── = عزل حراري سطح
│   │           └── = بلاط سطح
│   └── محيط ──→ وزرات سطح
│               └── = درابزين سطح
│
└── floor[BASEMENT]
    ├── area ──→ عزل مائي أساسات
    └── height ──→ لياسة بدروم
```

## 8. التوصيات الفنية للتنفيذ

### 8.1 توسيع بطاقة إعدادات المبنى

الحقول الإضافية المقترحة لإعدادات المبنى:

```typescript
interface ExtendedBuildingConfig {
  // الحقول الحالية
  totalLandArea: number;
  buildingPerimeter: number;
  floors: FloorConfig[];

  // حقول مقترحة جديدة
  landPerimeter?: number;         // محيط الأرض (للسور)
  fenceHeight?: number;           // ارتفاع السور (م)

  // بيانات إضافية لكل دور
  floors: ExtendedFloorConfig[];
}

interface ExtendedFloorConfig extends FloorConfig {
  // الحقول الحالية
  id: string;
  name: string;
  area: number;
  height: number;
  floorType: FloorType;
  isRepeated: boolean;
  repeatCount: number;

  // حقول مقترحة جديدة
  rooms?: RoomConfig[];           // تعريف الغرف في الدور
  bathroomCount?: number;         // عدد الحمامات
  bathroomTotalWallArea?: number; // مساحة جدران الحمامات الإجمالية
  openings?: OpeningConfig[];     // الفتحات (أبواب + نوافذ)
}

interface RoomConfig {
  id: string;
  name: string;
  length: number;
  width: number;
  area: number;            // = length × width
  type: 'bedroom' | 'living' | 'kitchen' | 'bathroom' | 'hall' | 'other';
  hasFalseCeiling: boolean;
}

interface OpeningConfig {
  id: string;
  type: 'door' | 'window';
  subType: string;         // باب عادي، باب حمام، نافذة كبيرة، ...
  width: number;
  height: number;
  count: number;
  isExternal: boolean;     // هل على الواجهة الخارجية؟
}
```

### 8.2 نظام الربط التلقائي (Auto-Linking)

#### المفهوم: "مصدر البيانات" (DataSource)

كل بند تشطيب يمكن أن يكون له مصدر بيانات:

```typescript
type DataSource =
  | { type: 'manual' }                          // إدخال يدوي (الوضع الحالي)
  | { type: 'building_config'; floorId: string } // مرتبط بإعدادات المبنى
  | { type: 'linked_item'; sourceItemId: string; transform?: Transform }  // مرتبط ببند آخر
  | { type: 'derived'; formula: string }         // محسوب بمعادلة

type Transform =
  | { op: 'equal' }                    // تساوي كامل
  | { op: 'subtract'; value: number }  // طرح قيمة
  | { op: 'multiply'; factor: number } // ضرب بمعامل
  | { op: 'subtract_item'; itemId: string } // طرح بند آخر
```

#### آلية العمل المقترحة

1. **عند إضافة بند جديد:**
   - خيار "ربط بإعدادات المبنى" → يختار الدور → تُحسب الكمية تلقائياً
   - خيار "ربط ببند آخر" → يختار البند المصدر → يحدد نوع العلاقة
   - خيار "إدخال يدوي" → الوضع الحالي

2. **عند تغيير إعدادات المبنى:**
   - تتحدث تلقائياً كل البنود المرتبطة بها
   - تظهر إشعار بالبنود المتأثرة

3. **عند تغيير بند مصدر:**
   - تتحدث تلقائياً كل البنود المرتبطة به
   - سلسلة التحديث تنتشر (cascade)

### 8.3 سيناريوهات الربط العملية

#### السيناريو 1: إعداد مبنى فيلا دورين + ملحق + سطح

```
إعدادات المبنى:
├── مساحة الأرض: 625 م² (25×25)
├── محيط المبنى: 50 م.ط (12.5×4)
├── الأدوار:
│   ├── أرضي: 300 م², ارتفاع 3.2م
│   ├── أول: 300 م², ارتفاع 3.2م
│   ├── ملحق: 120 م², ارتفاع 3.0م
│   └── سطح: 300 م², ارتفاع 0م
```

**البنود المشتقة تلقائياً:**

| البند | الدور | الكمية | المصدر |
|-------|-------|--------|--------|
| أرضيات بورسلان | أرضي | 300 م² | إعدادات: مساحة الأرضي |
| أرضيات بورسلان | أول | 300 م² | إعدادات: مساحة الأول |
| أرضيات سيراميك | ملحق | 120 م² | إعدادات: مساحة الملحق |
| لياسة داخلية | أرضي | ~480 م² | محيط الغرف × 3.2 - فتحات |
| لياسة داخلية | أول | ~480 م² | محيط الغرف × 3.2 - فتحات |
| دهان داخلي | أرضي | ~430 م² | = لياسة أرضي - جدران حمامات |
| دهان داخلي | أول | ~430 م² | = لياسة أول - جدران حمامات |
| بلاط جدران حمامات | أرضي | ~50 م² | = جدران حمامات أرضي |
| بلاط جدران حمامات | أول | ~50 م² | = جدران حمامات أول |
| أسقف مستعارة | أرضي | 300 م² | = أرضيات أرضي |
| أسقف مستعارة | أول | 300 م² | = أرضيات أول |
| لياسة خارجية | - | ~640 م² | 50 × (3.2+3.2+3.0) - فتحات |
| دهان واجهات | - | ~640 م² | = لياسة خارجية |
| عزل مائي سطح | - | 300 م² | إعدادات: مساحة السطح |
| عزل حراري سطح | - | 300 م² | = عزل مائي سطح |
| بلاط سطح | - | 300 م² | = عزل مائي سطح |
| أرضيات حوش | - | 325 م² | 625 - 300 |
| عزل حراري جدران | - | ~470 م² | 50 × 9.4 |

#### السيناريو 2: تغيير مساحة الدور الأرضي من 300 إلى 350 م²

**البنود التي تتحدث تلقائياً:**
1. أرضيات الأرضي: 300 → 350
2. أسقف مستعارة الأرضي: 300 → 350
3. لياسة داخلية الأرضي: تتغير تبعاً للمساحة الجديدة
4. دهان داخلي الأرضي: يتغير تبعاً
5. أرضيات الحوش: 325 → 275 (625-350)
6. عزل مائي سطح: قد يتغير إذا كان السطح مرتبطاً

### 8.6 ملاحظات هامة

1. **الأولوية في الربط:** إعدادات المبنى → اللياسة → باقي البنود. اللياسة هي "البند الأم" الذي تُشتق منه معظم البنود الأخرى.

2. **الحمامات كمفتاح:** تحديد عدد ومساحات الحمامات في إعدادات المبنى (أو في اللياسة) يُنتج تلقائياً:
   - بلاط جدران الحمامات
   - الفرق بين اللياسة والدهان
   - عزل مائي الحمامات
   - أرضيات الحمامات (سيراميك عادة)

3. **الفتحات كمفتاح:** تعريف الفتحات مرة واحدة (في اللياسة أو في إعدادات المبنى) يُنتج:
   - خصم الفتحات من اللياسة والدهان
   - عدد الأبواب الداخلية والخارجية
   - مساحات النوافذ
   - خصم الفتحات من الواجهات الحجرية

4. **السطح كوحدة متكاملة:** تعريف مساحة السطح مرة واحدة يُنتج تلقائياً 4-5 بنود (عزل مائي، عزل حراري، بلاط، وزرات، درابزين).

5. **التحديث التتابعي (Cascade):** عند تغيير قيمة في المصدر، يجب أن تنتشر التغييرات عبر السلسلة:
   ```
   إعدادات المبنى → لياسة → دهان → (تحديث التكاليف)
                   → أرضيات → أسقف → (تحديث التكاليف)
                   → عزل → (تحديث التكاليف)
   ```

6. **المرونة:** يجب دائماً السماح بالتعديل اليدوي وفك الربط، لأن بعض المشاريع لها حالات خاصة لا تنطبق عليها القواعد العامة.

## الخلاصة

النظام الحالي يعمل بشكل **مستقل لكل بند** - كل عنصر تشطيب يُدخل مساحاته وكمياته يدوياً دون أي ربط مع إعدادات المبنى أو البنود الأخرى. هذا يسبب:

1. **تكرار الإدخال:** نفس المساحات تُدخل مراراً في بنود مختلفة
2. **عدم الاتساق:** تغيير مساحة دور لا يؤثر على البنود المرتبطة
3. **أخطاء بشرية:** احتمال إدخال قيم مختلفة لنفس المساحة في بنود مختلفة

الحل المقترح يعتمد على **ثلاث طبقات من الربط:**
1. **إعدادات المبنى → البنود:** ربط مباشر بمساحات وارتفاعات الأدوار
2. **بند → بند:** ربط بين البنود المتساوية أو المشتقة
3. **تحديث تتابعي:** أي تغيير ينتشر تلقائياً عبر سلسلة العلاقات
