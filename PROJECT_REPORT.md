# تقرير شامل عن مشروع مسار (Masar)

---

## القسم الأول: تعريف المشروع

### ما هو مسار؟

مسار هو منصة متكاملة لإدارة المشاريع الإنشائية، مصممة كتطبيق ويب حديث يعمل كخدمة سحابية (SaaS). اسم "مسار" يعكس فلسفة التطبيق في توجيه المشاريع نحو الإنجاز عبر مسار واضح ومنظم. المنصة تستهدف شركات المقاولات والمكاتب الهندسية وفرق إدارة المشاريع الإنشائية، وتوفر لهم أدوات شاملة لإدارة كل جوانب المشروع من التخطيط إلى التسليم.

### ماذا يقدم مسار؟

المنصة تقدم مجموعة متكاملة من الخدمات تشمل:

**إدارة المشاريع الإنشائية** — إنشاء المشاريع وتتبعها، تعيين فرق العمل، تحديد الأدوار والصلاحيات، متابعة التقدم الفعلي مقابل المخطط، وإدارة الجداول الزمنية والمراحل الرئيسية.

**الإدارة المالية** — تتبع المصروفات حسب الفئات (مواد، عمالة، معدات، مقاولين باطن، نقل، متنوعة)، إدارة المطالبات والدفعات، إصدار الفواتير، متابعة التدفق النقدي، إدارة الحسابات البنكية، وإعداد التقارير المالية.

**التنفيذ الميداني** — تقارير يومية مع حالة الطقس، توثيق الصور بتصنيفات متعددة (تقدم، مشاكل، معدات، مواد، سلامة)، تتبع المشاكل حسب مستوى الخطورة، وتحديثات التقدم.

**إدارة المستندات** — نظام مجلدات (عقود، مخططات، مطالبات، خطابات، صور)، مرفقات، مشاركة المستندات، وسير عمل الاعتمادات متعدد المراحل.

**إدارة العقود والمقاولين** — عقود المقاولين من الباطن، أوامر التغيير، شروط الدفع، ومتابعة المستحقات.

**التواصل** — رسائل فريق العمل الداخلية، بوابة مالك المشروع للتواصل الرسمي، نظام الإشعارات، والملخصات الأسبوعية.

**إدارة الشركة** — إدارة الموظفين، تعيينات المشاريع، إدارة الأصول، كشوف الرواتب، ومصروفات الشركة.

**التسعير والعروض** — دراسات التكلفة، عروض الأسعار، حساب الكميات، والتسعير التفصيلي.

**بوابة المالك** — واجهة منفصلة لمالك المشروع للاطلاع على التحديثات والتقارير والتواصل مع فريق الإدارة.

**الذكاء الاصطناعي** — مساعد ذكي (Chatbot) يساعد في الاستفسارات والتحليلات.

---

### نظام الأدوار والصلاحيات

المنصة تعتمد نظام صلاحيات هرمي دقيق:

- **MANAGER (مدير المشروع)** — صلاحيات كاملة على كل جوانب المشروع
- **ENGINEER (مهندس)** — صلاحيات التنفيذ والمتابعة الفنية
- **SUPERVISOR (مشرف)** — صلاحيات الإشراف الميداني
- **ACCOUNTANT (محاسب)** — صلاحيات مالية ومحاسبية
- **VIEWER (مشاهد)** — صلاحيات قراءة فقط

كل قسم في الواجهة يظهر أو يختفي بناءً على دور المستخدم، مما يضمن أن كل شخص يرى فقط ما يحتاجه.

---

### البنية التقنية

المشروع مبني كـ Monorepo باستخدام pnpm و Turborepo، ويتكون من:

**التطبيق الرئيسي (apps/web):**
- Next.js مع App Router — إطار عمل الواجهة الأمامية
- React — مكتبة واجهة المستخدم
- TypeScript — لضمان أمان الأنواع
- Tailwind CSS — تنسيق الواجهة مع دعم كامل لاتجاه RTL (العربية)
- shadcn/ui + Radix UI — مكونات واجهة المستخدم
- Recharts — الرسوم البيانية والمخططات

**الحزم المشتركة (packages/):**
- api — طبقة API باستخدام ORPC + Hono.js (استدعاءات إجرائية آمنة الأنواع)
- database — Prisma ORM مع PostgreSQL (أكثر من 85 جدول)
- auth — نظام المصادقة باستخدام Better Auth
- i18n — الترجمة باستخدام next-intl (عربي، إنجليزي، ألماني)
- payments — بوابات الدفع (Stripe وغيرها)
- storage — تخزين الملفات عبر AWS S3
- ai — ميزات الذكاء الاصطناعي
- mail — خدمة البريد الإلكتروني

**إدارة الحالة:**
- TanStack React Query v5 — لإدارة البيانات من الخادم
- ORPC — لاستدعاءات API آمنة الأنواع من البداية للنهاية

## القسم الثاني: الداشبورد الرئيسية (لوحة التحكم)

### نظرة عامة

الداشبورد الرئيسية هي الصفحة الأولى التي يراها المستخدم عند فتح أي مشروع. تعرض ملخصاً شاملاً لحالة المشروع من حيث التنفيذ والمالية والجدول الزمني، مع إجراءات سريعة وأنشطة حديثة.

### التسلسل الهرمي للصفحة

عندما يفتح المستخدم داشبورد مشروع، يمر الطلب عبر عدة طبقات:

1. **Layout الأساسي** — يتحقق من الجلسة وحالة الاشتراك
2. **Layout المنظمة** — يتحقق من صلاحيات المنظمة ويوفر السياق
3. **Layout المشروع** — يجلب بيانات المشروع، يحدد دور المستخدم، ويغلف المحتوى بـ ProjectShell
4. **صفحة المشروع** — تحمل مكون ProjectOverview ديناميكياً

### الهيكل المرئي للداشبورد

الداشبورد تتكون من شبكة بطاقات (Cards) مرتبة كالتالي:

**الصف الأول — ثلاثة أعمدة رئيسية:**

- **بطاقة مراحل التنفيذ (ExecutionPhasesCard)** — تعرض مخطط دائري (Donut) لتوزيع حالات المراحل (مكتمل، قيد التنفيذ، لم يبدأ)، نسبة الإنجاز الكلية في المنتصف، قائمة المراحل مع أشرطة التقدم، وشرائح صحة المشروع (على المسار / معرض للخطر / متأخر).

- **بطاقة الميزانية والمالية (FinanceBudgetCard)** — تعرض أربعة مؤشرات أداء رئيسية: المصروفات الفعلية، الدفعات المستلمة، المتبقي، والاحتباس. شريط تقدم استخدام الميزانية، حسابات الربح والتدفق النقدي، وتقدير الدفعة القادمة.

- **بطاقة الجدول الزمني (TimelineScheduleCard)** — منحنى S يقارن التقدم المخطط مع الفعلي على مدار 12 شهراً، شريط التقدم مع نقطة مؤشر، حساب الأيام المتبقية، وحالة المشروع (على المسار أو معرض للخطر مع نقطة متحركة).

**الصف الثاني — إجراءات سريعة (QuickActions):**

أربعة أزرار ملونة في شبكة رباعية:
- تقرير يومي (أزرق) — ينقل لصفحة إنشاء تقرير جديد
- رفع صورة (بنفسجي) — ينقل لصفحة رفع الصور
- إضافة مصروف (أخضر) — ينقل لصفحة إضافة مصروف جديد
- الإبلاغ عن مشكلة (كهرماني) — ينقل لصفحة إنشاء بلاغ

هذه الأزرار تُفلتر حسب دور المستخدم.

**الصف الثالث — مخطط الجدول الزمني (ProjectTimelineChart):**

يظهر بعرض كامل لمن لديهم صلاحية مشاهدة قسم الميدان. يعرض تصور بصري للمراحل والمعالم.

**الصف الرابع — عمودين:**

- **مخطط فئات المصروفات (ExpenseCategoryChart)** — مخطط دائري يوضح توزيع المصروفات حسب الفئة (مواد، عمالة، معدات، مقاولين باطن، نقل، متنوعة).

- **مستندات سريعة (QuickDocumentsCard)** — يظهر لمن لديهم صلاحية المستندات، يعرض آخر المستندات مع وصول سريع.

**الصف الخامس — ثلاثة أعمدة:**

- **نبض النشاط (ActivityPulseCard)** — مؤشر حالة النشاط الميداني (يظهر لمن لديهم صلاحية الميدان).

- **الأنشطة الأخيرة (RecentActivitiesCard)** — جدول زمني لآخر 3 أنشطة ميدانية (تقارير يومية، مشاكل، صور، تحديثات تقدم) مع نقاط ملونة وأوقات نسبية ورابط "عرض الكل".

### نظام التنقل (Navigation)

#### الهيدر (ProjectHeader)

يتضمن:
- زر العودة لقائمة المشاريع
- اسم المشروع وتحية المستخدم
- ساعة حقيقية تتحدث كل دقيقة
- زر التحديثات (ينقل لصفحة /updates)
- محدد العرض حسب الدور (ViewAsSelector) للمديرين
- شريط العقد (ContractBar) يعرض مؤشرات مالية

#### شريط التنقل للكمبيوتر (DesktopNavBar)

يظهر على الشاشات المتوسطة وما فوق، بتصميم زجاجي (Glassmorphism) مع أزرار حبوب أفقية:

- نظرة عامة (رابط مباشر)
- التنفيذ (رابط مباشر)
- المقاولين (رابط مباشر)
- المصروفات (رابط مباشر)
- المدفوعات (رابط مباشر)
- المساحة/المستندات (رابط مباشر)
- المزيد (قائمة منبثقة تضم: المالك، التحليلات، الفريق)

الأقسام المرئية تتحدد حسب دور المستخدم.

#### شريط التنقل للموبايل (MobileBottomNav)

شريط ثابت أسفل الشاشة (z-index: 50) مع أيقونات وتسميات:
- روابط مباشرة للأقسام الفردية
- قوائم منبثقة (Popover) للأقسام المجمعة
- أوراق منزلقة (Sheet) للمجموعات الكبيرة
- مؤشرات بصرية للحالة النشطة (لون أساسي + نقطة)

---

### الغلاف الرئيسي (ProjectShell)

مكون العميل الرئيسي الذي يغلف كل صفحات المشروع. هيكله:

```
ProjectRoleProvider (سياق الأدوار)
  ├── ProjectHeader (الهيدر مع زر العودة والعنوان والساعة)
  ├── ProjectNavigation (تنقل الكمبيوتر + الموبايل)
  ├── منطقة المحتوى الرئيسية (قابلة للتمرير)
  └── FloatingChatButton (زر المحادثة العائم)
```

يدعم اتجاه RTL بالكامل (`dir="rtl"`) وتنسيق التواريخ بالعربية.

## القسم الرابع: دليل التعديل

### كيف تعدل ترتيب البطاقات في الداشبورد؟

افتح ملف `ProjectOverview.tsx` وعدّل ترتيب المكونات داخل JSX. البطاقات مرتبة في شبكة CSS Grid، يمكنك تغيير ترتيبها أو تعديل أعمدة الشبكة.

### كيف تضيف بطاقة جديدة؟

1. أنشئ مكون جديد في مجلد `overview/` (مثال: `NewCard.tsx`)
2. استورده في `ProjectOverview.tsx`
3. أضفه في المكان المناسب ضمن شبكة البطاقات
4. إذا كانت البطاقة تحتاج بيانات، أنشئ endpoint جديد في `packages/api/`

### كيف تعدل التنقل (أضف قسم جديد)؟

1. افتح ملف `constants.ts` في مجلد `shell/`
2. أضف القسم الجديد في مصفوفة `PROJECT_NAV_SECTIONS`
3. حدد مجموعته في `NAV_GROUP_CONFIG`
4. حدد الأدوار المسموح لها بمشاهدته

### كيف تعدل الهيدر؟

عدّل ملف `ProjectHeader.tsx` لتغيير محتوى الهيدر (العنوان، الأزرار، الساعة، شريط العقد).

### كيف تعدل التصميم الزجاجي (Glassmorphism)؟

عدّل الثوابت في `constants.ts` — خصوصاً `GLASS_CLASSES` لتغيير الشفافية والضبابية والحدود والظلال.

### كيف تعدل البيانات المعروضة في بطاقة معينة؟

كل بطاقة تجلب بياناتها عبر ORPC hook. مثلاً:
- بيانات المالية ← عدّل `packages/api/modules/project-finance/router.ts`
- بيانات الجدول الزمني ← عدّل `packages/api/modules/project-timeline/router.ts`
- بيانات الميدان ← عدّل `packages/api/modules/project-field/router.ts`

### كيف تعدل الصلاحيات (من يرى ماذا)؟

عدّل ملف `use-project-role` في مجلد hooks لتغيير قواعد الرؤية. دالة `canViewSection` هي التي تتحكم في إظهار أو إخفاء الأقسام.

### كيف تعدل ألوان حالات المشروع؟

الألوان موزعة في ملفات البطاقات نفسها:
- مكتمل: أخضر زمردي (Emerald)
- قيد التنفيذ: أزرق (Blue)
- متأخر: أحمر (Red)
- لم يبدأ: رمادي (Slate)
- على المسار: أخضر
- معرض للخطر: كهرماني (Amber)

## القسم السادس: قاعدة البيانات

قاعدة البيانات تحتوي على أكثر من 85 جدول تغطي جميع جوانب المنصة:

**الجداول الرئيسية:**
- User, Organization, Member, Role — المستخدمين والمنظمات
- Project, ProjectMember — المشاريع وأعضائها
- ProjectDailyReport, ProjectPhoto, ProjectIssue — التنفيذ الميداني
