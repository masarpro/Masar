# المرحلة 2 — "المحاسبة التلقائية" — برومبت تنفيذي لـ Claude Code

## نظرة عامة

هذا أكبر تغيير معماري في النظام المالي لمسار. نضيف **طبقة محاسبية كاملة** فوق النظام الحالي بدون كسر أي شيء موجود.

**المبدأ الأساسي:** المقاول يستمر يستخدم مسار بالضبط مثل اليوم — الفرق أن كل عملية مالية تولّد قيد محاسبي في الخلفية تلقائياً. المحاسب يرى القيود ويراجعها. المقاول بدون محاسب يستفيد من التقارير المحاسبية بدون ما يعرف يعني إيش قيد مزدوج.

**مقسّم إلى 5 مراحل فرعية:**
1. Schema changes (نماذج + enums + migration)
2. دليل الحسابات (seed + CRUD + UI)
3. بنية القيود اليومية (model + manual entries + UI)
4. محرك التوليد التلقائي (hooks على كل العمليات المالية)
5. التقارير المحاسبية (ميزان المراجعة + الميزانية العمومية)

### الخطوة 3: إضافة نموذج Account (دليل الحسابات)

```prisma
model Account {
  id              String        @id @default(cuid())
  organizationId  String
  
  code            String        // "1100", "5200" — رمز الحساب
  nameAr          String        // الاسم بالعربية
  nameEn          String        // الاسم بالإنجليزية
  description     String?       // وصف اختياري
  
  type            AccountType   // ASSET, LIABILITY, EQUITY, REVENUE, EXPENSE
  normalBalance   NormalBalance // DEBIT أو CREDIT
  level           Int           // 1=رئيسي, 2=فرعي, 3=تفصيلي, 4=تحليلي
  
  parentId        String?       // الحساب الأب (شجرة هرمية)
  parent          Account?      @relation("AccountHierarchy", fields: [parentId], references: [id])
  children        Account[]     @relation("AccountHierarchy")
  
  isSystem        Boolean       @default(false) // حسابات النظام لا تُحذف
  isActive        Boolean       @default(true)
  isPostable      Boolean       @default(true)  // هل يمكن الترحيل عليه مباشرة؟ (false للحسابات الرئيسية)
  
  // ربط اختياري بحساب بنكي
  bankAccountId   String?       @unique // ربط 1-1 مع OrganizationBank
  
  // العلاقات
  organization    Organization  @relation(fields: [organizationId], references: [id], onDelete: Cascade)
  journalLines    JournalEntryLine[]
  
  createdAt       DateTime      @default(now())
  updatedAt       DateTime      @updatedAt
  
  @@unique([organizationId, code])
  @@index([organizationId, type])
  @@index([organizationId, parentId])
  @@index([organizationId, isActive])
}
```

### الخطوة 3: دالة seedChartOfAccounts

في نفس الملف `accounting.ts`، أضف:

```typescript
export async function seedChartOfAccounts(
  db: PrismaClient,
  organizationId: string
): Promise<{ created: number; skipped: number }> {
  // 1. تحقق: هل يوجد حسابات مسبقة؟
  const existingCount = await db.account.count({ where: { organizationId } });
  if (existingCount > 0) {
    return { created: 0, skipped: existingCount };
  }

  // 2. أنشئ الحسابات على مرحلتين (الآباء أولاً ثم الأبناء)
  //    لأن parentId يحتاج أن يكون الأب موجوداً
  
  // المرحلة أ: أنشئ كل الحسابات بدون parentId
  const accountMap = new Map<string, string>(); // code → id
  
  for (const acc of DEFAULT_CHART_OF_ACCOUNTS) {
    const created = await db.account.create({
      data: {
        organizationId,
        code: acc.code,
        nameAr: acc.nameAr,
        nameEn: acc.nameEn,
        type: acc.type,
        normalBalance: acc.normalBalance,
        level: acc.level,
        isSystem: acc.isSystem,
        isPostable: acc.isPostable,
        // parentId يُضاف في المرحلة الثانية
      },
    });
    accountMap.set(acc.code, created.id);
  }
  
  // المرحلة ب: ربط الآباء
  for (const acc of DEFAULT_CHART_OF_ACCOUNTS) {
    if (acc.parentCode) {
      const parentId = accountMap.get(acc.parentCode);
      const accountId = accountMap.get(acc.code);
      if (parentId && accountId) {
        await db.account.update({
          where: { id: accountId },
          data: { parentId },
        });
      }
    }

  const bankAccounts = await db.organizationBank.findMany({
    where: { organizationId },
    select: { id: true, name: true },
  });
  
  const cashParentId = accountMap.get("1110"); // النقدية والبنوك
  if (cashParentId && bankAccounts.length > 0) {
    for (let i = 0; i < bankAccounts.length; i++) {
      const bankCode = `111${(i + 1).toString().padStart(1, "0")}`; // 1111, 1112, ...
      const bankAccount = await db.account.create({
        data: {
          organizationId,
          code: bankCode,
          nameAr: bankAccounts[i].name,
          nameEn: bankAccounts[i].name,
          type: "ASSET",
          normalBalance: "DEBIT",
          level: 4,
          parentId: cashParentId,
          isSystem: false,
          isPostable: true,
          bankAccountId: bankAccounts[i].id,
        },
      });
      
      // ربط البنك بالحساب المحاسبي
      await db.organizationBank.update({
        where: { id: bankAccounts[i].id },
        data: { accountId: bankAccount.id },
      });
    }
  }
```

```typescript
import { type PrismaClient, Prisma } from "@prisma/client";
import { createJournalEntry } from "@repo/database/queries/accounting";

// ========================================
// دالة مساعدة: هل المحاسبة مفعّلة لهذه المنظمة؟
// ========================================
async function isAccountingEnabled(db: PrismaClient, organizationId: string): Promise<boolean> {
  const count = await db.account.count({ where: { organizationId } });
  return count > 0;
}

// ========================================
// دالة مساعدة: جلب حساب بالرمز
// ========================================
async function getAccountByCode(db: PrismaClient, organizationId: string, code: string): Promise<string | null> {
  const account = await db.account.findUnique({
    where: { organizationId_code: { organizationId, code } },
    select: { id: true },
  });
  return account?.id ?? null;
}

// ========================================
// دالة مساعدة: جلب حساب البنك المحاسبي
// ========================================
async function getBankAccountId(db: PrismaClient, organizationId: string, bankId: string): Promise<string | null> {
  const bank = await db.organizationBank.findUnique({
    where: { id: bankId },
    select: { accountId: true },
  });
  // إذا البنك غير مربوط بحساب محاسبي، استخدم حساب "النقدية والبنوك" العام
  if (bank?.accountId) return bank.accountId;
  return getAccountByCode(db, organizationId, "1110");
}

// ========================================
// 1. إصدار فاتورة → مدين: العملاء / دائن: الإيرادات + ضريبة
// ========================================
export async function onInvoiceIssued(db: PrismaClient, invoice: {
  id: string;
  organizationId: string;
  number: string;
  issueDate: Date;
  clientName: string;
  totalAmount: Prisma.Decimal;
  vatAmount: Prisma.Decimal;
  projectId?: string | null;
}) {
  if (!(await isAccountingEnabled(db, invoice.organizationId))) return;

  const receivableId = await getAccountByCode(db, invoice.organizationId, "1120");
  const revenueId = await getAccountByCode(db, invoice.organizationId, "4100");
  const vatPayableId = await getAccountByCode(db, invoice.organizationId, "2130");
  
  if (!receivableId || !revenueId || !vatPayableId) return; // حسابات مفقودة — skip silently

  const netAmount = invoice.totalAmount.sub(invoice.vatAmount);
  const lines: any[] = [
    { accountId: receivableId, debit: invoice.totalAmount, credit: new Prisma.Decimal(0), description: `فاتورة ${invoice.number} — ${invoice.clientName}`, projectId: invoice.projectId },
    { accountId: revenueId, debit: new Prisma.Decimal(0), credit: netAmount, description: `إيراد فاتورة ${invoice.number}`, projectId: invoice.projectId },
  ];

  // إضافة بند الضريبة فقط إذا > 0
  if (invoice.vatAmount.greaterThan(0)) {
    lines.push({ accountId: vatPayableId, debit: new Prisma.Decimal(0), credit: invoice.vatAmount, description: `ضريبة فاتورة ${invoice.number}` });
  }

  await createJournalEntry(db, {
    organizationId: invoice.organizationId,
    date: invoice.issueDate,
    description: `إصدار فاتورة ${invoice.number} — ${invoice.clientName}`,
    referenceType: "INVOICE",
    referenceId: invoice.id,
    referenceNo: invoice.number,
    isAutoGenerated: true,
    lines,
  });
}

  const subAccId = await getAccountByCode(db, payment.organizationId, "5200");
  const bankAccId = await getBankAccountId(db, payment.organizationId, payment.sourceAccountId);
  if (!subAccId || !bankAccId) return;

  await createJournalEntry(db, {
    organizationId: payment.organizationId,
    date: payment.date,
    description: `دفعة مقاول باطن — ${payment.contractorName}`,
    referenceType: "SUBCONTRACT_PAYMENT",
    referenceId: payment.id,
    isAutoGenerated: true,
    lines: [
      { accountId: subAccId, debit: payment.amount, credit: new Prisma.Decimal(0), projectId: payment.projectId },
      { accountId: bankAccId, debit: new Prisma.Decimal(0), credit: payment.amount },
    ],
  });
}

// ========================================
// 6. اعتماد مسير رواتب → مدين: الرواتب / دائن: البنك
// ========================================
export async function onPayrollApproved(db: PrismaClient, payroll: {
  id: string;
  organizationId: string;
  month: number;
  year: number;
  totalNet: Prisma.Decimal;
  totalGosi: Prisma.Decimal;
  sourceAccountId?: string | null;
}) {
  if (!(await isAccountingEnabled(db, payroll.organizationId))) return;

  const salaryAccId = await getAccountByCode(db, payroll.organizationId, "6100");
  const gosiAccId = await getAccountByCode(db, payroll.organizationId, "2170");
  let bankAccId: string | null = null;
  if (payroll.sourceAccountId) {
    bankAccId = await getBankAccountId(db, payroll.organizationId, payroll.sourceAccountId);
  } else {
    bankAccId = await getAccountByCode(db, payroll.organizationId, "1110");
  }
  
  if (!salaryAccId || !bankAccId) return;

  const lines: any[] = [
    { accountId: salaryAccId, debit: payroll.totalNet.add(payroll.totalGosi), credit: new Prisma.Decimal(0), description: `رواتب ${payroll.month}/${payroll.year}` },
    { accountId: bankAccId, debit: new Prisma.Decimal(0), credit: payroll.totalNet, description: `صرف رواتب ${payroll.month}/${payroll.year}` },
  ];

  // إضافة بند التأمينات إذا > 0
  if (gosiAccId && payroll.totalGosi.greaterThan(0)) {
    lines.push({ accountId: gosiAccId, debit: new Prisma.Decimal(0), credit: payroll.totalGosi, description: `تأمينات اجتماعية ${payroll.month}/${payroll.year}` });
  }

  await createJournalEntry(db, {
    organizationId: payroll.organizationId,
    date: new Date(payroll.year, payroll.month - 1, 28), // آخر الشهر تقريباً
    description: `مسير رواتب ${payroll.month}/${payroll.year}`,
    referenceType: "PAYROLL",
    referenceId: payroll.id,
    isAutoGenerated: true,
    lines,
  });
}

// ========================================
// 7. مقبوضات مباشرة (FinancePayment) → مدين: البنك / دائن: إيرادات
// ========================================
export async function onOrganizationPaymentReceived(db: PrismaClient, payment: {
  id: string;
  organizationId: string;
  amount: Prisma.Decimal;
  date: Date;
  description: string;
  destinationAccountId: string;
  projectId?: string | null;
}) {
  if (!(await isAccountingEnabled(db, payment.organizationId))) return;

  const bankAccId = await getBankAccountId(db, payment.organizationId, payment.destinationAccountId);
  const revenueAccId = await getAccountByCode(db, payment.organizationId, "4300");
  if (!bankAccId || !revenueAccId) return;

  await createJournalEntry(db, {
    organizationId: payment.organizationId,
    date: payment.date,
    description: `مقبوضات: ${payment.description}`,
    referenceType: "ORG_PAYMENT",
    referenceId: payment.id,
    isAutoGenerated: true,
    lines: [
      { accountId: bankAccId, debit: payment.amount, credit: new Prisma.Decimal(0), projectId: payment.projectId },
      { accountId: revenueAccId, debit: new Prisma.Decimal(0), credit: payment.amount, projectId: payment.projectId },
    ],
  });
}

// ========================================
// 8. إشعار دائن → مدين: الإيرادات + الضريبة / دائن: العملاء
// ========================================
export async function onCreditNoteIssued(db: PrismaClient, creditNote: {
  id: string;
  organizationId: string;
  number: string;
  issueDate: Date;
  clientName: string;
  totalAmount: Prisma.Decimal;
  vatAmount: Prisma.Decimal;
  projectId?: string | null;
}) {
  if (!(await isAccountingEnabled(db, creditNote.organizationId))) return;

  const receivableId = await getAccountByCode(db, creditNote.organizationId, "1120");
  const revenueId = await getAccountByCode(db, creditNote.organizationId, "4100");
  const vatPayableId = await getAccountByCode(db, creditNote.organizationId, "2130");
  if (!receivableId || !revenueId || !vatPayableId) return;

  const netAmount = creditNote.totalAmount.sub(creditNote.vatAmount);
  const lines: any[] = [
    { accountId: revenueId, debit: netAmount, credit: new Prisma.Decimal(0), description: `إشعار دائن ${creditNote.number}`, projectId: creditNote.projectId },
    { accountId: receivableId, debit: new Prisma.Decimal(0), credit: creditNote.totalAmount, projectId: creditNote.projectId },
  ];

  if (creditNote.vatAmount.greaterThan(0)) {
    lines.push({ accountId: vatPayableId, debit: creditNote.vatAmount, credit: new Prisma.Decimal(0) });
  }

  await createJournalEntry(db, {
    organizationId: creditNote.organizationId,
    date: creditNote.issueDate,
    description: `إشعار دائن ${creditNote.number} — ${creditNote.clientName}`,
    referenceType: "CREDIT_NOTE",
    referenceId: creditNote.id,
    referenceNo: creditNote.number,
    isAutoGenerated: true,
    lines,
  });
}
```

### الخطوة 3: ربط المحرك بالعمليات الحالية

**مهم جداً:** هذه الخطوة تتطلب تعديل الـ procedures المالية الحالية. التعديل يكون **إضافة سطر واحد** بعد نجاح العملية — لا تغيير في المنطق الحالي أبداً.

**النمط:**
```typescript
// في كل procedure مالي، بعد نجاح العملية:
try {
  await onInvoiceIssued(db, { ...invoiceData });
} catch (e) {
  // صامت — لا تفشل العملية الأصلية بسبب خطأ محاسبي
  console.error("[AutoJournal] Failed to generate entry:", e);
}
```

**الملفات التي تحتاج تعديل (أضف سطر واحد في كل منها):**

| العملية | الملف المتوقع | الدالة | يُستدعى |
|---------|-------------|--------|---------|
| إصدار فاتورة | `invoice-issue.ts` أو `finance.ts` | بعد تغيير الحالة لـ ISSUED | `onInvoiceIssued` |
| دفعة على فاتورة | `invoice-add-payment.ts` | بعد إنشاء FinanceInvoicePayment | `onInvoicePaymentReceived` |
| إنشاء إشعار دائن | `invoice-create-credit-note.ts` | بعد إنشاء الإشعار | `onCreditNoteIssued` |
| إنشاء/دفع مصروف | `org-finance.ts` createExpense أو payExpense | بعد الإنشاء بحالة COMPLETED | `onExpenseCompleted` |
| تحويل بين حسابات | `org-finance.ts` createTransfer | بعد إنشاء التحويل | `onTransferCompleted` |
| دفعة مقاول باطن | `subcontract-payments.ts` | بعد إنشاء الدفعة | `onSubcontractPayment` |
| اعتماد رواتب | `payroll.ts` approve | بعد تغيير الحالة لـ APPROVED | `onPayrollApproved` |
| مقبوضات مباشرة | `org-finance.ts` createPayment | بعد إنشاء المقبوض | `onOrganizationPaymentReceived` |

**⚠️ اقرأ كل ملف أولاً** لتحديد الموقع الصحيح لإضافة الاستدعاء. ابحث عن النقطة بعد نجاح العملية وقبل `return`.

# المرحلة الفرعية 5/5: التقارير المحاسبية (ميزان المراجعة + ميزانية عمومية)

## الهدف
بناء تقريرين محاسبيين أساسيين يُستخرجان من القيود المرحّلة.

## التعليمات

### الخطوة 1: ميزان المراجعة (Trial Balance)

أضف في `packages/database/prisma/queries/accounting.ts`:

```typescript
export interface TrialBalanceRow {
  accountId: string;
  accountCode: string;
  accountNameAr: string;
  accountNameEn: string;
  accountType: AccountType;
  level: number;
  debitBalance: Prisma.Decimal;
  creditBalance: Prisma.Decimal;
}

export interface TrialBalanceResult {
  rows: TrialBalanceRow[];
  totals: { totalDebit: Prisma.Decimal; totalCredit: Prisma.Decimal };
  isBalanced: boolean; // يجب أن يكون true دائماً
  asOfDate: Date;
}
```

**المنطق:**
- لكل حساب `isPostable = true`: مجموع المدين ومجموع الدائن من `JournalEntryLine` حيث `JournalEntry.status = POSTED` و `date <= asOfDate`
- إذا الحساب `normalBalance = DEBIT`: الرصيد = مجموع المدين - مجموع الدائن (إذا موجب → debitBalance، إذا سالب → creditBalance)
- إذا `normalBalance = CREDIT`: العكس
- الإجماليات يجب أن تتساوى (debit = credit)

**UI:** جدول بسيط — رمز الحساب | الاسم | مدين | دائن. صف إجمالي في الأسفل مُميّز.

### الخطوة 2: الميزانية العمومية (Balance Sheet)

```typescript
export interface BalanceSheetResult {
  asOfDate: Date;
  assets: {
    currentAssets: { accounts: AccountBalanceRow[]; total: Prisma.Decimal };
    fixedAssets: { accounts: AccountBalanceRow[]; total: Prisma.Decimal };
    totalAssets: Prisma.Decimal;
  };
  liabilities: {
    currentLiabilities: { accounts: AccountBalanceRow[]; total: Prisma.Decimal };
    totalLiabilities: Prisma.Decimal;
  };
  equity: {
    accounts: AccountBalanceRow[];
    currentYearPL: Prisma.Decimal; // إيرادات - مصروفات للعام الحالي
    totalEquity: Prisma.Decimal;
  };
  isBalanced: boolean; // الأصول = الخصوم + حقوق الملكية
}
```

**المنطق:**
- الأصول = كل حسابات `type = ASSET` بأرصدتها
- الخصوم = كل حسابات `type = LIABILITY`
- حقوق الملكية = حسابات `type = EQUITY` + (إيرادات - مصروفات) للعام الحالي
- أرباح/خسائر العام = مجموع حسابات `REVENUE` - مجموع حسابات `EXPENSE` للسنة المالية الحالية

**UI:** تنسيق محاسبي كلاسيكي — قسمين: الأصول على اليمين (RTL)، الخصوم + حقوق الملكية على اليسار. صف "الأصول = الخصوم + حقوق الملكية" في الأسفل.

# ملاحظات عامة حرجة لكل المراحل

## 1. التوافق مع النظام الحالي
- **القاعدة #1:** لا تكسر أي عملية مالية موجودة. التوليد التلقائي يعمل في try/catch صامت.
- **القاعدة #2:** المحاسبة اختيارية. إذا لم يُفعّل المستخدم "تفعيل المحاسبة"، كل شيء يعمل كالسابق.
- **القاعدة #3:** لا تغيير في سلوك الفواتير/المصروفات/المدفوعات — فقط إضافة قيد في الخلفية.

## 2. أنماط يجب اتباعها
- كل query يبدأ بـ `organizationId` (multi-tenancy)
- حقول مالية: `Prisma.Decimal` فقط — `@db.Decimal(15, 2)`
- Mutations: `subscriptionProcedure` (ليس `protectedProcedure`)
- RTL: `ms-`, `me-`, `ps-`, `pe-`
- ترجمة: `ar.json` + `en.json` بالتوازي
- بعد أي تغيير في schema: `pnpm --filter database db:generate`
- بعد كل مرحلة: `pnpm tsc --noEmit && pnpm build`

## 3. ترتيب التنفيذ المقترح
```
المرحلة 1/5 (Schema)      → db:generate → tsc → commit
المرحلة 2/5 (دليل حسابات) → tsc → build → commit  
المرحلة 3/5 (القيود)       → tsc → build → commit
المرحلة 4/5 (التوليد)      → tsc → build → commit ← الأهم والأخطر
المرحلة 5/5 (التقارير)     → tsc → build → commit
```
