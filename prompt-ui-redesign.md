# برومبت إعادة تصميم واجهات التسعير — مسار

> **الهدف:** إعادة تصميم الواجهات المرئية لقسم التسعير بحسب الـ wireframes في خطة إعادة الهيكلة.
> **السياق:** المكونات V2 موجودة ومربوطة بالصفحات، لكن شكلها المرئي لم يتغير عن القديم.
> **كل فيز مستقل** — يمكن تنفيذه في جلسة Claude Code منفصلة.

---

## التعليمات العامة (تُطبّق في كل فيز)

- ابدأ في **Plan Mode** — اقرأ الملف المستهدف بالكامل قبل أي تعديل
- RTL / Arabic-first UI — كل النصوص عربية
- استخدم shadcn/ui components (Card, Tabs, Collapsible, Badge, Button, Table, etc.)
- استخدم Tailwind CSS — لا inline styles
- استخدم lucide-react للأيقونات
- next-intl للترجمة (useTranslations)
- لا `@ts-ignore` ولا `any` جديدة (الموجودة مقبولة مؤقتاً)
- بعد كل تعديل: `npx tsc --noEmit --pretty 2>&1 | grep "[FILENAME]" | grep -v "TS2307" | head -20`
- **لا تُغيّر الـ data fetching أو الـ mutations** — فقط عدّل العرض المرئي (JSX + Tailwind)
- إذا احتجت بيانات غير متوفرة حالياً، اعرض placeholder/skeleton بدلاً من إضافة API calls


### التصميم المطلوب:

شريط أفقي يظهر أعلى كل صفحة داخل دراسة. لكل مرحلة:

```
┌──────────────────────────────────────────────────────────────────────────┐
│  ✅ الكميات  →  🔵 المواصفات  →  ⚪ التكلفة  →  ⚪ التسعير  →  ⚪ عرض السعر  │
│  45 بند         جاري العمل       مقفل           مقفل           مقفل         │
│  معتمد ✓        12 من 45                                                    │
└──────────────────────────────────────────────────────────────────────────┘
```

#### المتطلبات:
- **مرحلة مكتملة (APPROVED):** خلفية خضراء فاتحة، أيقونة ✓ خضراء، نص "معتمد" + عدد البنود
- **مرحلة نشطة (IN_PROGRESS):** خلفية زرقاء فاتحة (primary)، نقطة نبض (pulse)، نص "جاري العمل" + نسبة الإنجاز
- **مرحلة مقفلة (LOCKED):** خلفية رمادية (muted)، أيقونة قفل، نص "مقفل"، `cursor-not-allowed opacity-50`
- **مرحلة في الانتظار (NOT_STARTED):** خلفية بيضاء بحدود، لا أيقونة خاصة
- بين كل مرحلة سهم أو خط فاصل (`→` أو `ChevronLeft` لـ RTL)
- الضغط على مرحلة مكتملة → ينقل للصفحة (router.push)
- الضغط على مرحلة مقفلة → لا يفعل شيئاً
- استجابة responsive — في الموبايل تظهر أيقونات فقط بدون نص

## ملاحظات حرجة

1. **كل فيز مستقل** — إذا نفد الـ context، أكمل في جلسة جديدة من الفيز التالي
2. **لا تمسّ الـ data layer** — فقط عدّل JSX و Tailwind و imports لمكونات shadcn
3. **إذا وجدت أن المكون يحتاج data غير متوفرة:** اعرض placeholder مع نص "سيتوفر قريباً" بدلاً من إضافة API calls
4. **اقرأ الملف الكامل قبل التعديل** — لا تفترض الهيكل الحالي
5. **الأولوية:** الفيز 1 (شريط المراحل + نظرة عامة) > الفيز 3 (تسعير التكلفة) > الفيز 4 (التسعير) > الفيز 2 (الكميات) > الباقي
6. **إذا الملف طويل جداً (>500 سطر):** عدّل section بـ section — لا تعيد كتابة الملف بالكامل
7. **بعد كل فيز:** اعرض ملخص مختصر لما تغيّر واسأل "هل أتابع للفيز التالي؟"
