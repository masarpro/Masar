## الفلسفة الأساسية للتصميم الجديد

### المبدأ الأول: فصل المسؤوليات (Separation of Concerns)

كل مرحلة تعمل كوحدة مستقلة تمامًا. الكميات لا تعرف شيئًا عن المواصفات. المواصفات لا تعرف شيئًا عن الأسعار. كل مرحلة تُصدّر "جدولًا" نظيفًا تستقبله المرحلة التالية.

### المبدأ الثاني: نقاط دخول متعددة (Multiple Entry Points)

المستخدم لا يبدأ دائمًا من الصفر. أحيانًا لديه كميات جاهزة، أحيانًا لديه كميات ومواصفات، أحيانًا لديه عقد كامل. النظام يسمح بالقفز لأي مرحلة مع استيراد البيانات.

### المبدأ الثالث: فصل الأدوار (Role Separation)

مهندس الكميات ≠ مهندس المواصفات ≠ المُسعّر ≠ المدير. كل شخص يرى فقط ما يخصه ويعمل في مرحلته.

### المبدأ الرابع: المرونة في الحجم (Scale Flexibility)

من بند واحد إلى مشروع ضخم — نفس الواجهة، نفس التدفق، بدون تعقيد زائد.

## السيناريوهات الـ 8 لنقاط الدخول

### السيناريو 1: مشروع جديد من الصفر
```
نقطة الدخول → الكميات (حساب من الصفر)
المسار: الكميات → المواصفات → تسعير التكلفة → التسعير → عرض السعر → مشروع
```

### السيناريو 2: لدي كميات جاهزة وأريد تحديد المواصفات
```
نقطة الدخول → المواصفات (استيراد كميات)
المسار: [استيراد كميات] → المواصفات → تسعير التكلفة → التسعير → عرض السعر → مشروع
```

### السيناريو 3: لدي كميات ومواصفات وأريد التسعير
```
نقطة الدخول → تسعير التكلفة (استيراد كميات + مواصفات)
المسار: [استيراد] → تسعير التكلفة → التسعير → عرض السعر → مشروع
```

### السيناريو 4: لدي تسعير تكلفة وأريد عرض سعر
```
نقطة الدخول → التسعير النهائي (استيراد التكاليف)
المسار: [استيراد] → التسعير → عرض السعر → مشروع
```

### السيناريو 5: لدي عقد متكامل وأريد تحويله لمشروع
```
نقطة الدخول → تحويل لمشروع (استيراد عقد)
المسار: [استيراد عقد] → مشروع مباشرة
```

### السيناريو 6: مشروع بالمقطوعية وأريد دراسة التكلفة
```
نقطة الدخول → الكميات (مع تحديد قيمة العقد المقطوعة)
المسار: الكميات → المواصفات → تسعير التكلفة → مقارنة مع المقطوعية → مشروع
الهدف: معرفة هل المقطوعية مربحة أم لا
```

### السيناريو 7: بنود مخصصة (بند واحد أو عدة بنود)
```
نقطة الدخول → الكميات (وضع البنود المخصصة)
المسار: الكميات → المواصفات → تسعير التكلفة → التسعير → عرض السعر
مثال: عميل يريد فقط تسعير أعمال الدهان لفيلا
```

### السيناريو 8: تسعير سريع (بدون تفاصيل)
```
نقطة الدخول → التسعير السريع
المسار: إدخال مباشر للبنود والأسعار → عرض السعر
مثال: عميل يتصل ويريد سعر تقريبي فوري
```

## القسم 1: الكميات (Quantities)

### الغرض
حساب الكميات الفعلية للمشروع — بدون أي مواصفات أو أسعار.

### ما يحتويه هذا القسم فقط
- الأبعاد والمساحات والأحجام
- عدد العناصر (أبواب، نوافذ، نقاط كهرباء...)
- أوزان (حديد التسليح)
- الكميات المحسوبة بالوحدات الهندسية (م³، م²، م.ط، كجم، عدد)

### ما لا يحتويه هذا القسم
- ❌ لا مواصفات مواد
- ❌ لا أسعار
- ❌ لا تكاليف
- ❌ لا جداول مواد (BOM)

## ملخص التغييرات عن النظام الحالي

### ما سيُحذف
1. ❌ بطاقة "ملخص التكاليف" من صفحة الكميات
2. ❌ بطاقة "ملخص الكميات" القديمة (تُستبدل بملخصات متخصصة)
3. ❌ اختيار المواصفات من صفحة الكميات (يُنقل للمواصفات)
4. ❌ جدول المواد (BOM) من صفحة الكميات (يُنقل للمواصفات)
5. ❌ التبويبات الثابتة (إنشائي/تشطيبات/MEP/يدوي) — تُعرض فقط المختارة

### ما سيُنقل
1. 🔄 المواصفات → من الكميات إلى قسم المواصفات المستقل
2. 🔄 جدول المواد (BOM) → من الكميات إلى مُخرجات المواصفات
3. 🔄 الأسعار → من المواصفات إلى تسعير التكلفة فقط
4. 🔄 هوامش الربح → من تسعير التكلفة إلى التسعير النهائي (قسم المدير)

### ما سيُضاف
1. ✅ شاشة إنشاء ذكية بنقاط دخول متعددة
2. ✅ ملخص إنشائي شامل (خرسانة + حديد + بلوك + قص + توزيع)
3. ✅ طرق عرض متعددة (بحسب الدور / البند / المشروع)
4. ✅ تخصيص مواصفات لغرفة/دور محدد
5. ✅ سيناريوهات عمالة متعددة (وحدة/مسطح/مقطوعية/راتب/مختلط)
6. ✅ 4 أنماط لعرض السعر (تفصيلي/متر مسطح/مقطوعية/بسيط)
7. ✅ تخصيص الأعمدة والصفوف في عرض السعر
8. ✅ تحليل المقطوعية (مقارنة التكلفة المحسوبة مع قيمة العقد)
9. ✅ إدخال سريع (Quick Entry) للمستخدم الخبير
10. ✅ وضع البطاقات القابلة للتوسيع بدلاً من الجداول المزدحمة

### ما سيبقى كما هو (محسّن)
1. 🟢 محرك الاشتقاق الذكي (التشطيبات + MEP)
2. 🟢 معالج إعداد المبنى (4 خطوات)
3. 🟢 خوارزمية القص والتوزيع
4. 🟢 الحسابات الإنشائية المتقدمة
5. 🟢 كتالوج المواصفات والقوالب
6. 🟢 نظام المراحل والصلاحيات

## أولويات التنفيذ المقترحة

### المرحلة 1 — إعادة الهيكلة الأساسية (2-3 أسابيع)
1. فصل قسم المواصفات عن الكميات
2. فصل التسعير النهائي عن تسعير التكلفة
3. إنشاء شاشة الإنشاء الذكية بنقاط الدخول
4. تحديث الشريط الجانبي وشريط المراحل

### المرحلة 2 — تحسين الكميات (1-2 أسبوع)
1. ملخص الإنشائي الشامل (خرسانة/حديد/قص/بلوك)
2. طرق العرض المتعددة للتشطيبات
3. ملخصات مُجمّعة لكل قسم
4. حذف البيانات غير المنتمية (أسعار/مواصفات/تكاليف)

### المرحلة 3 — تحسين المواصفات (1-2 أسبوع)
1. واجهة اختيار المواصفات الجديدة
2. تخصيص مواصفات بحسب الدور/الغرفة
3. جدول المواد المُجمّع (BOM) بطرق عرض مختلفة
4. القوالب السريعة

### المرحلة 4 — تحسين تسعير التكلفة (2 أسابيع)
1. واجهة إدخال الأسعار الجديدة (بطاقات + إدخال سريع)
2. سيناريوهات العمالة المتعددة
3. دعم المقطوعية لكل بند
4. ملخص التكاليف المحسّن

### المرحلة 5 — التسعير وعرض السعر (1-2 أسبوع)
1. واجهة التسعير النهائي (هوامش متعددة)
2. أنماط عرض السعر الـ 4
3. تخصيص الأعمدة والصفوف
4. معاينة وتصدير

### المرحلة 6 — السيناريوهات المتقدمة (1 أسبوع)
1. تحليل المقطوعية
2. استيراد الكميات/المواصفات من ملفات
3. التسعير السريع
4. نقاط الدخول المتعددة
