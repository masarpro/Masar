## الملخص التنفيذي

هذا البرومت يغطي بناء **وحدة إدارة المشتريات (Procurement Module)** لمنصة مسار — الفجوة الحرجة رقم 1 في تحليل الفجوات. الوحدة تغطي دورة المشتريات الكاملة التي يتوقعها المقاول السعودي:

```
طلب شراء (PR) → طلب عروض أسعار (RFQ) → أمر شراء (PO) → استلام بضاعة (GR) → فاتورة مورد → دفع
```

### نقاط التكامل مع الوضع الحالي لمسار:
- **المشاريع (Project):** كل طلب شراء مرتبط بمشروع محدد
- **المالية (FinanceExpense):** فاتورة المورد تُنشئ `FinanceExpense` تلقائياً
- **الحسابات البنكية (OrganizationBank):** الدفع يخصم من حساب بنكي
- **المصروفات (ProjectExpense):** ربط اختياري بمصروف المشروع
- **التسعير (CostStudy):** مقارنة أسعار الشراء مع تقديرات دراسة التكلفة
- **المقاولات (SubcontractContract):** مشتريات لصالح مقاولي الباطن
- **سجل التدقيق (OrgAuditAction):** تسجيل كل عملية في سجل المنظمة
- **التسلسلات (OrganizationSequence):** أرقام تلقائية PR-001, PO-001, إلخ
- **الصلاحيات (permissions.ts):** قسم `procurement` جديد في مصفوفة RBAC

### المرحلة 3.4: نمط الـ Procedure (مثال — إنشاء طلب شراء)

```typescript
// packages/api/modules/procurement/procedures/purchase-requests/create.ts

import { protectedProcedure } from \"../../../../orpc/procedures\";
import { z } from \"zod\";
import { db } from \"@repo/database\";

export const prCreate = protectedProcedure
  .input(z.object({
    organizationId: z.string(),
    projectId: z.string(),
    title: z.string().min(1),
    description: z.string().optional(),
    priority: z.enum([\"LOW\", \"MEDIUM\", \"HIGH\", \"URGENT\"]).default(\"MEDIUM\"),
    requiredDate: z.string().datetime().optional(),
    notes: z.string().optional(),
    items: z.array(z.object({
      name: z.string().min(1),
      description: z.string().optional(),
      unit: z.string().min(1),
      quantity: z.number().positive(),
      estimatedPrice: z.number().min(0).default(0),
      costStudyItemId: z.string().optional(),
      costStudyItemType: z.string().optional(),
    })).min(1),
  }))
  .handler(async ({ input, context }) => {
    const { user } = context;
    
    // 1. التحقق من العضوية والصلاحيات
    // (اتبع نمط subcontracts/create.ts للتحقق)
    
    // 2. توليد الرقم التسلسلي
    // استخدم OrganizationSequence مع sequenceKey = \"PR\"
    // نفس نمط generateContractNo في subcontracts
    
    // 3. حساب الإجمالي التقديري
    const estimatedTotal = input.items.reduce(
      (sum, item) => sum + (item.quantity * item.estimatedPrice), 0
    );
    
    // 4. إنشاء طلب الشراء مع البنود
    const pr = await db.purchaseRequest.create({
      data: {
        organizationId: input.organizationId,
        projectId: input.projectId,
        prNumber: generatedNumber, // PR-001
        title: input.title,
        description: input.description,
        priority: input.priority,
        requiredDate: input.requiredDate ? new Date(input.requiredDate) : undefined,
        estimatedTotal,
        requestedById: user.id,
        createdById: user.id,
        notes: input.notes,
        items: {
          create: input.items.map((item, index) => ({
            name: item.name,
            description: item.description,
            unit: item.unit,
            quantity: item.quantity,
            estimatedPrice: item.estimatedPrice,
            totalEstimate: item.quantity * item.estimatedPrice,
            costStudyItemId: item.costStudyItemId,
            costStudyItemType: item.costStudyItemType,
            sortOrder: index,
          })),
        },
      },
      include: { items: true, project: true },
    });
    
    // 5. تسجيل في سجل التدقيق
    await db.organizationAuditLog.create({
      data: {
        organizationId: input.organizationId,
        actorId: user.id,
        action: \"PR_CREATED\",
        entityType: \"PurchaseRequest\",
        entityId: pr.id,
        metadata: { prNumber: pr.prNumber, projectId: input.projectId },
      },
    });
    
    return pr;
  });
```

3. عند اعتماد كل بنود PR → حالة PR تصبح FULLY_ORDERED

8. كل عملية → تسجيل في OrganizationAuditLog
9. كل رقم تسلسلي → عبر OrganizationSequence (sequenceKey: PR, RFQ, PO, GR, VI)
